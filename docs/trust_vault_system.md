# Proof Vault / Trust Layer

## Purpose
Prove control, not profits. Every artifact shows monitoring, discipline, and system quality.

## Folder structure (Google Drive, Dropbox, or Supabase Storage)
```
Angel Precision Proof Vault
├── 01 Execution Screenshots
├── 02 Health Dashboard Screenshots
├── 03 Ops Summaries
├── 04 Rejection Saves
├── 05 System Protections
├── 06 Bug Fixes
├── 07 Monitored Exits
├── 08 Operator Reports
├── 09 Trade Lifecycle Examples
└── 10 Approved Sales Assets
```

## Naming format
```
YYYY-MM-DD_artifact-type_short-description_sensitivity
```

Examples:
```
2026-05-08_health-dashboard_scanner-runtime-visible_internal
2026-05-08_rejection-save_bad-contract-filtered_internal
2026-05-08_operator-report_daily-health-summary_sales-approved
```

## Sensitivity levels
- `internal` — for your records only
- `content-approved` — safe to use in posts
- `sales-approved` — safe to show on calls

## Daily target
Save at least ONE proof artifact per day:
- health dashboard screenshot
- rejection example
- operator report
- bug fix note
- monitored exit
- risk-control screenshot

## How to use on calls
When a lead asks "how do I know this is controlled?" — you don't scramble.
You pull from the vault and show evidence of monitoring, rejection logic, and operator oversight.

## What never goes in the vault
- P&L screenshots promising returns
- cherry-picked winners
- anything that implies guaranteed performance
